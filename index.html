Magic Number Method Guide

Step-by-Step Process:

  1. Identify the "Interesting Octet": Find the octet in the subnet mask that is neither 0 nor 255.
  2. Calculate the Magic Number: Magic Number = 256 - Interesting Octet Value
  3. Apply the Rules:
    • If mask octet = 255: Copy the IP address octet value exactly
    • If mask octet = 0: Use 0 for subnet ID, 255 for broadcast address
    • For the interesting octet: Use magic number multiples
  4. Find the Subnet Range: Determine which magic number multiple the IP address falls into
  5. Calculate Addresses:
    • Subnet ID: Start of the range
    • Broadcast: End of the range
    • First Host: Subnet ID + 1
    • Last Host: Broadcast - 1

Example Problems

1-Octet Mask Example

IP: 10.180.122.244

Mask: 255.248.0.0 (/13)

Magic Number: 256 - 248 = 8

Result: Subnet 10.176.0.0 - 10.183.255.255

2-Octet Mask Example

IP: 165.245.77.14

Mask: 255.255.240.0 (/20)

Magic Number: 256 - 240 = 16

Result: Subnet 165.245.64.0 - 165.245.79.255

3-Octet Mask Example

IP: 172.16.24.133

Mask: 255.255.255.224 (/27)

Magic Number: 256 - 224 = 32
